# Declaración del Equipo del Proyecto

## Roles y Responsabilidades
Considerando el ambiente académico del Taller de Proyectos 2 y el marco Scrum, el equipo asume los siguientes perfiles de responsabilidad y contribución compartida:

1. **Scrum Master / Analista:**
   - **Responsabilidades:** Velar por la aplicación de metodologías ágiles. Asegurar la recolección, documentación, y validación continua de requisitos y supuestos de los sistemas complejos. Mantener sincronizado y documentado el entorno de GitHub.

2. **Product Owner / Arquitecto de Software:**
   - **Responsabilidades:** Establecer la visión del producto final y tomar las decisiones tecnológicas y arquitectónicas. Representar los intereses académicos exigidos en las restricciones del problema definiendo las prioridades en el _Product Backlog_.

3. **Software Engineer / Full-Stack Developer:**
   - **Responsabilidades:** Desarrollar tanto la SPA del Frontend como las lógicas e integraciones API REST en el Backend con NodeJS. Codificar estructurar los estilos y la UI Premium. Embeber las consideraciones de OWASP y código limpio.

4. **Algorithms Engineer (Especialista en Optimización):**
   - **Responsabilidades:** Investigar, diseñar de manera independiente y escribir el código lógico puro del Constraint Satisfaction Problem (CSP) y sus subrutinas de Backtracking o propación de restricciones.

*(Nota: Dada la naturaleza de los proyectos universitarios y la formación integral, todos los miembros colaborarán y asumirán roles cruzados, fusionando el análisis investigativo con el desarrollo de alta calidad).*

## Normas de Trabajo (Working Agreements)
1. **Comunicación:** Todo avance, rediseño o cambio en la arquitectura debe discutirse y documentarse, usando GitHub.
2. **Control de Versiones:** Estricto flujo de repositorios. Las integraciones seguirán la metodología de ramas para características nuevas (Feature branches).
3. **Calidad de Entregable:** Los algoritmos base deben contar con cobertura de código e integrarse con prácticas ecológicas y de bajo costo transaccional (Green Software) en la medida de lo computacionalmente permisible.
